# Iris Decision Tree MLOps Demo Report

## 1. Project Purpose

This project demonstrates a small but complete MLOps workflow using MLflow.

The goal is not only to train a model, but to make every training run traceable:

```text
new data -> preprocessing -> data version -> model training -> evaluation -> MLflow tracking -> model artifact
```

The current model is a pure Python decision tree classifier for Iris flower classification.

## 2. Current Project Structure

```text
iris_mlflow_project/
```

## 3. Data Generation Logic

Raw data is stored in:

```text
data/raw/iris_events.csv
```

Each training run can append new rows to this raw dataset.

The synthetic data is generated around three class profiles:

```text
setosa      -> [5.0, 3.4, 1.5, 0.25]
versicolor  -> [6.1, 2.8, 4.4, 1.35]
virginica   -> [6.6, 3.0, 5.6, 2.05]
```

Each generated row is created as:

```text
class profile + random noise
```

Each row also includes:

```text
event_id
event_date
label
```

A small percentage of feature values are intentionally left blank to simulate messy real-world data.

## 4. Preprocessing Logic

Preprocessing is implemented in:

```text
src/data.py
```

The preprocessing process does the following:

1. Load raw rows from `data/raw/iris_events.csv`.
2. Compute median values for the original numeric features.
3. Fill missing values with the median.
4. Create engineered features.
5. Split the processed data into train and test sets.
6. Generate a data version hash.
7. Save the processed dataset under `data/versions/<data_version>/`.

Original features:

```text
sepal_length
sepal_width
petal_length
petal_width
```

Engineered features:

```text
petal_ratio = petal_length / petal_width
sepal_ratio = sepal_length / sepal_width
petal_area  = petal_length * petal_width
sepal_area  = sepal_length * sepal_width
```

Final model features:

```text
sepal_length
sepal_width
petal_length
petal_width
petal_ratio
sepal_ratio
petal_area
sepal_area
```

## 5. Data Versioning

A data version is generated from:

```text
processed rows + preprocessing configuration
```

The project computes:

```text
data_version = first 12 chars of SHA256(processed_data + preprocess_config)
```

This means the data version changes when any of these changes:

```text
raw data content
number of rows
missing value handling
feature engineering logic
train/test split ratio
random seed
feature list
```

## 6. Training Logic

Training is run from:

```text
train.py
```

Example command:

```powershell
.venv\Scripts\python.exe train.py --add-new-data 100 --max-depth 5
```

Meaning:

```text
use the project virtual environment
run train.py
append 100 new raw data rows
train a decision tree with max_depth = 5
log the run to MLflow
```

The model is implemented in:

```text
src/model.py
```

The model type is:

```text
DecisionTreeClassifier
```

It is a pure Python implementation, not sklearn. This avoids native DLL issues on locked-down Windows machines.

## 8. Artifacts

In MLflow, artifacts are files generated by a training run.

They are different from parameters and metrics:

```text
Parameter = small configuration value, e.g. max_depth = 5
Metric    = numeric result, e.g. accuracy = 0.9765
Artifact  = file, e.g. model file, confusion.json, train.csv
```

This project logs these artifacts:

```text
decision_tree.json
confusion.json
data/manifest.json
data/train.csv
data/test.csv
model artifacts
```

Artifact meanings:

```text
decision_tree.json
  Human-readable decision tree structure.

confusion.json
  Shows actual label vs predicted label counts.

manifest.json
  Describes the data version used by the run.

train.csv / test.csv
  Exact preprocessed data used in that run.

model artifacts
  MLflow-loadable model files such as MLmodel and python_model.pkl.
```

Artifacts are stored in:

```text
artifacts/
```

MLflow metadata is stored in:

```text
mlflow.db
```

## 9. Confusion Matrix Meaning

`confusion.json` records how predictions were distributed.

Example format:

```json
{
  "setosa predicted_as setosa": 30,
  "versicolor predicted_as virginica": 2
}
```

Meaning:

```text
actual setosa was predicted as setosa 30 times
actual versicolor was predicted as virginica 2 times
```

Accuracy tells the overall correctness.

Confusion matrix tells where the model is wrong.

## 11. How This Demonstrates MLOps Value

Without MLOps, a model is just a file and a score.

With MLflow, each model answers:

```text
Which data version trained this model?
How many raw rows were used?
How many rows were newly added?
What preprocessing was applied?
What features were used?
What model parameters were used?
What was the accuracy?
Where is the model artifact?
Can another person reproduce the run?
```

This is the practical value of MLOps:

```text
traceability
reproducibility
model comparison
data lineage
artifact management
model versioning
handover clarity
```
